The Core Objective

Citizen Developer → Full-Stack BTP Developer

SAP FusionForge is not just a template. It's a learning accelerator. Every agent output is designed to be readable, understandable, and educational.

🎯

Imagine hiring a whole company of AI workers.

Each AI worker has a specific job title, knows their responsibilities, and works together as a team to build software — just like a real SAP consulting company.

🏢 What is Paperclip?

Think of it like a company operating system. Just like a real company has org charts, budgets, and job roles — Paperclip lets you define all of that for AI agents. Each agent has a job title, a monthly budget, and tasks to complete.

🔧 What is SAP BTP?

Think of it like a magical toolkit for building business software. SAP BTP (Business Technology Platform) is a cloud platform where companies build apps, automate processes, and connect their systems.

🤝 What is a Fusion Team?

Imagine a kitchen with two types of cooks. Expert chefs (pro-code developers) and home cooks (citizen developers who use drag-and-drop tools). A Fusion Team combines both. The experts write the engine. The home cooks build the screens. Together they deliver faster.

"If an AI agent is an employee, Paperclip is the company."

The Team

Meet the AI Agents

Each agent is like a specialist employee. They each have a specific job, know when to pass work to the next person, and never overstep their role.

🚀 Start Here
📋
Business Analyst
Entry point · Citizen Dev role

The first agent to run. Reads the client config file, understands the business problem, and writes clear user stories (requirement cards) for the rest of the team.

🧒 Simple: "Like a student who listens to what the teacher wants and writes it down clearly."

Reads CLIENT.md Writes User Stories GitHub Issues
🔍 SA's PA · New
🔍
Discovery Advisor
SA Personal Assistant · Research

Runs in parallel with the Solution Architect. Searches SAP Discovery Center for pre-built missions that match the client scenario — so the SA doesn't design what SAP already built.

🧒 Simple: "Like a researcher who checks the library before the architect draws anything."

3-Tier Intel: Catalog match → Live web search → Redesign advisory
Discovery Center Web Intel Alignment Report
🏛️
Solution Architect
Hub · Orchestrator · Pro-code

The team lead. Reads user stories from the BA plus the Discovery Alignment Report from the Discovery Advisor — then designs the system and writes ADRs that reference existing SAP missions where possible.

🧒 Simple: "Like the project manager who makes the blueprint before anyone starts building."

BTP Landscape ADRs Mermaid
⚙️
CAP Developer
Backend services · Pro-code

Writes the backend engine. Creates SAP CAP projects with data models and OData APIs that other apps connect to. Uses PostgreSQL locally for demos.

🧒 Simple: "Like the mechanic who builds the engine under the hood."

CAP + OData PostgreSQL BTP CF
🎨
Build Apps Developer
Frontend apps · Low-code

Designs the user-facing app screens. Connects them to the CAP backend using OData. Uses SAP UI5 Web Components — the same visual building blocks SAP Build Apps uses.

🧒 Simple: "Like the interior designer who makes the house look nice and functional."

UI5 Components OData App Specs
🔄
BPA Designer
Process automation · Low-code

Designs approval workflows and automated processes. Produces BPMN 2.0 XML — a universal standard that works in SAP Build Process Automation AND open-source Camunda.

🧒 Simple: "Like drawing the traffic rules — who approves what, and in what order."

BPMN 2.0 Camunda SAP BPA
🚀
BTP Admin
Platform operations · Pro-code

Manages the SAP BTP platform itself. Writes scripts to set up subaccounts, assign entitlements, and configure connections. Includes Docker Compose for local demos.

🧒 Simple: "Like the building manager who sets up the electricity and plumbing before anyone moves in."

BTP CLI Docker Config

Comms

How agents talk to each other

Three shared files act as the team's communication layer.

🧠

MEMORY.md

Shared long-term context. Every agent reads this at the start of every run. Holds what has been built, decisions made, and active handoffs.

💬

TEAM_LOG.md

The WhatsApp group. Agents append status updates here — what they did, what's next, blockers, and handoffs. Full audit trail.

📢

GitHub Discussions

Public noticeboard. Solution Architect posts weekly sprint summaries here. Anyone visiting the repo can see what's being built.

State

One database for the whole team

A single SQLite file shared across all agents. Structured, queryable state.

Table Purpose
user_stories BA writes every user story here. Full status tracking from draft to done.
decisions SA writes ADRs here. Every architecture decision is searchable.
artifacts Every file produced by every agent is logged — type, path, version.
handoffs Structured version of TEAM_LOG.md handoffs. Agents query pending handoffs.
cost_log Every LLM API call is logged with token count and cost.
skill_runs Every skill run tracked — pending, running, done, failed.
